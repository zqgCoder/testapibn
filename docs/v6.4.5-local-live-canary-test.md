# v6.4.5 Local Live Canary Test (Preflight Only)

本版在 v6.4.4 Live Canary Safety Guards 基础上，增加 **本地 live canary 测试前的 preflight 与控制面展示**。

**本阶段不执行真实实盘交易**，不提供一键实盘下单按钮。

## 目标

- 在 `/dashboard/runtime-control` 展示 Live Canary Preflight 面板
- 汇总 live guard、Runtime、BTCUSDT 市场状态、reconcile 结果
- 计算 `canary_ready` 与 `blocking_reasons`
- 为后续极小金额 live canary 手动流程提供检查清单

## API

```http
GET /dashboard/api/live-canary/preflight?token=YOUR_DASHBOARD_TOKEN
```

需 `DASHBOARD_TOKEN`（query 或 `X-Dashboard-Token`）。**不返回任何 secret**（含 `LIVE_CONFIRM_PHRASE` 原文）。

### 主要字段

| 字段 | 说明 |
|------|------|
| `binance_env` / `is_live` | 当前 Binance 环境 |
| `live_guard` | v6.4.4 live guard 状态快照 |
| `live_trading_enabled` | 是否允许 live 执行 |
| `live_confirm_phrase_configured` | 是否配置了确认短语 |
| `live_confirm_phrase_valid` | 是否与期望值匹配 |
| `live_allowed_symbols` / `live_max_*` | canary 限额配置 |
| `runtime` | locked / effective_locked / one_shot |
| `btcusdt` | 持仓、条件单、普通委托摘要 |
| `reconcile_summary` | 最近 reconcile summary |
| `canary_ready` | 是否满足 preflight 全部条件 |
| `blocking_reasons` | 阻塞原因代码列表 |
| `phase` | 固定 `preflight_only` |

## `canary_ready=true` 条件

1. `is_live=true`
2. live guard active
3. `LIVE_TRADING_ENABLED=true`
4. `LIVE_CONFIRM_PHRASE` 已配置且匹配 `LIVE_EXPECTED_CONFIRM_PHRASE`
5. `BTCUSDT` ∈ `LIVE_ALLOWED_SYMBOLS`
6. `LIVE_MAX_RISK_USDT` ≤ 1
7. `LIVE_MAX_MARGIN_USDT` ≤ 20
8. `LIVE_MAX_POSITION_NOTIONAL_USDT` ≤ 100
9. `LIVE_REJECT_TRADINGVIEW_BY_DEFAULT=true`
10. Runtime `locked=true`
11. one_shot `enabled=false`
12. BTCUSDT 无持仓、无条件单、无普通委托
13. reconcile `error_count=0`

## `blocking_reasons` 代码

| 代码 | 含义 |
|------|------|
| `not_live_environment` | 非 live endpoint |
| `live_guard_inactive` | live guard 未激活 |
| `live_trading_disabled` | `LIVE_TRADING_ENABLED=false` |
| `live_confirm_phrase_invalid` | 确认短语缺失或不匹配 |
| `btcusdt_not_allowed` | BTCUSDT 不在白名单 |
| `live_max_risk_too_high` | risk 上限 > 1 |
| `live_max_margin_too_high` | margin 上限 > 20 |
| `live_max_notional_too_high` | notional 上限 > 100 |
| `tradingview_live_not_rejected` | 未默认拒绝 TV |
| `runtime_not_locked` | Runtime 未锁定 |
| `one_shot_active` | one-shot 仍启用 |
| `btcusdt_position_not_flat` | 有持仓 |
| `btcusdt_algo_orders_exist` | 有条件单 |
| `btcusdt_open_orders_exist` | 有普通委托 |
| `reconcile_error` | reconcile 有 error |

## Dashboard

`/dashboard/runtime-control?token=...` 顶部 **Live Canary Preflight** 面板：

- `canary_ready` 徽章
- `blocking_reasons` 列表
- live 环境红色警告（仅 preflight，无下单按钮）
- demo 环境提示「当前不是 live，不能执行 live canary」

## 后续手动 live canary 流程（本版未实现）

1. 确认 preflight `canary_ready=true`
2. 人工 double-check `.env` 与 Binance 账户
3. Runtime one-shot unlock（单条信号）
4. 通过受控非-TV 渠道或后续版本允许的入口发送 **极小 risk** 信号
5. 验证 journal / 保护单 / 自动回锁
6. close → cleanup → reconcile
