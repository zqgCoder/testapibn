# v6.4.4 Live Canary Safety Guards

本版在 Binance **live/mainnet endpoint** 上新增多层 hard guard。**不修改 demo 环境行为**，且本版默认 **不真正执行实盘交易**。

## 配置（`.env`）

| 变量 | 默认 | 说明 |
|------|------|------|
| `LIVE_TRADING_ENABLED` | `false` | 必须为 `true` 才允许 live 执行 |
| `LIVE_CONFIRM_PHRASE` | 空 | 必须等于 `LIVE_EXPECTED_CONFIRM_PHRASE` |
| `LIVE_EXPECTED_CONFIRM_PHRASE` | `I_UNDERSTAND_THIS_IS_REAL_MONEY` | 期望确认短语 |
| `LIVE_ALLOWED_SYMBOLS` | `BTCUSDT` | 白名单 |
| `LIVE_MAX_RISK_USDT` | `1` | 信号/计划 risk 上限 |
| `LIVE_MAX_MARGIN_USDT` | `20` | margin 上限 |
| `LIVE_MAX_POSITION_NOTIONAL_USDT` | `100` | 计划名义价值上限 |
| `LIVE_REQUIRE_ONE_SHOT` | `true` | 必须有 Runtime one-shot |
| `LIVE_REJECT_TRADINGVIEW_BY_DEFAULT` | `true` | 默认拒绝 TV 来源 |
| `LIVE_FORCE_RUNTIME_LOCKED_ON_STARTUP` | `true` | live 启动强制 locked |
| `LIVE_CANARY_MODE` | `true` | canary 模式标记（Dashboard 展示） |

## 生效条件

仅当 `binance_env_label(BINANCE_BASE_URL) == "live"` 时，`app/live_guard.py` 参与校验。

Demo/testnet（`demo-fapi` / `testnet`）**完全跳过** Live Guard。

## 拒绝规则（live only）

1. `LIVE_TRADING_ENABLED != true` → `live_guard_trading_disabled`
2. 确认短语不匹配 → `live_guard_confirm_phrase_invalid`
3. symbol 不在白名单 → `live_guard_symbol_not_allowed`
4. `risk_usdt` 超限 → `live_guard_risk_too_large`
5. `margin_usdt` 超限 → `live_guard_margin_too_large`
6. 计划 `notional` 超限 → `live_guard_notional_too_large`
7. 无 one-shot → `live_guard_one_shot_required`
8. TradingView 来源 → `live_guard_tradingview_rejected`

Journal：`status=live_guard_rejected`，`skip_reason=live_guard_xxx`，`error_message` 含拒绝说明。

## 启动行为

`LIVE_FORCE_RUNTIME_LOCKED_ON_STARTUP=true` 且 endpoint 为 live 时，服务启动调用 `runtime_control.lock(reason="live canary startup safety lock")`。

## Dashboard

- `GET /dashboard/api/live-guard/status` — 返回 guard 状态（不含任何 secret）
- `/dashboard/runtime-control` — live 环境强警告横幅
- 主 Dashboard 风控配置 — live guard 检查项

## 离线测试（不连 Binance、不读真实 `.env`、不发网络请求）

仅验证 `app/live_guard.py` 拒绝逻辑，使用内存构造的 `Settings` / `TradingViewSignal` / `TradePlan`。

覆盖场景：

| # | 场景 | 期望 `skip_reason` |
|---|------|-------------------|
| 1 | `binance_env=demo` | 不阻断（`guard_active=false`） |
| 2 | live + `LIVE_TRADING_ENABLED=false` | `live_guard_trading_disabled` |
| 3 | 确认短语错误 | `live_guard_confirm_phrase_invalid` |
| 4 | symbol 非白名单 | `live_guard_symbol_not_allowed` |
| 5 | `risk_usdt` 超限 | `live_guard_risk_too_large` |
| 6 | `margin_usdt` 超限 | `live_guard_margin_too_large` |
| 7 | `source=tradingview` | `live_guard_tradingview_rejected` |
| 8 | 无 one-shot | `live_guard_one_shot_required` |
| 9 | plan notional 超限 | `live_guard_notional_too_large` |

### 运行方式

```bash
# 方式 A：unittest（推荐，无额外依赖）
python -m unittest tests.test_live_guard -v

# 方式 B：验证脚本（同上测试套件）
python scripts/verify_live_guard.py

# 方式 C：discover
python -m unittest discover -s tests -p "test_live_guard.py" -v
```

全部通过后再做 compile / import 检查：

```bash
python -m compileall main.py app tests scripts
python -c "from app.trader import Trader; print('import ok')"
```
