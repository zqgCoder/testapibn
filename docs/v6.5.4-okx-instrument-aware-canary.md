# v6.5.4 OKX Instrument-Aware Canary Guard

v6.5.3 允许在 sizing 不可行时仍生成 payload，POST 后 journal 失败。本版在 **payload 生成阶段** 与 **只读预检脚本** 中引入 instrument-aware 硬拒绝。

## 问题背景

BTC-USDT-SWAP 最小 1 张合约在 BTC ≈ 60,000 USDT、`ctVal=0.01` 时：

```
required_notional ≈ 1 × 0.01 × 60000 = 600 USDT
required_margin   ≈ 600 / leverage
```

若 `margin_usdt=20`、`leverage=1`，则 **600 > 20**，canary 不可行。v6.5.3 脚本仍写出 payload，导致 POST 后失败。

## 硬性规则（payload 脚本）

`scripts/build_local_okx_live_canary_payload.py` 在写文件前：

1. 从 OKX public API 读取 instrument metadata（`lotSz` / `minSz` / `ctVal`）
2. 用 `--close` 作为 mark price 估算最小 `sz` / notional / margin
3. 以下任一条件成立则 **拒绝生成**（exit code ≠ 0，**不创建** payload 文件）：
   - sizing 计算失败（如 `required_margin > margin_usdt`）
   - `required_notional > OKX_MAX_NOTIONAL_USDT`（`OKX_MAX_POSITION_NOTIONAL_USDT`）
4. 不打印 `WEBHOOK_SECRET` 或 payload 原文

拒绝时 stderr 输出：

| 字段 | 说明 |
|------|------|
| `instId` | 如 `BTC-USDT-SWAP` |
| `close` | 参考 mark/close 价格 |
| `minSz` / `lotSz` / `ctVal` | instrument metadata |
| `required_notional` | 最小张数对应名义价值 |
| `required_margin` | 最小张数对应保证金（notional/leverage） |
| `configured_margin_usdt` | `--margin-usdt` 或 payload 预算 |
| `configured_max_notional` | `.env` 中 `OKX_MAX_NOTIONAL_USDT` |

## 只读预检脚本

```bash
python scripts/check_okx_instrument_canary_feasibility.py
```

功能：

- 读取 OKX public instrument + live mark price
- 对照 `.env` 的 `OKX_MAX_MARGIN_USDT`、`OKX_MAX_NOTIONAL_USDT`、`--leverage`
- **不下单**，不打印 secret
- 输出 `canary_feasible=true/false` 与 `canary_suitability=SUITABLE|NOT_SUITABLE`

可选覆盖：

```bash
python scripts/check_okx_instrument_canary_feasibility.py --margin-usdt 20 --max-notional 100 --leverage 1
```

## BTC-USDT-SWAP 不适合 20 USDT canary 时

**不要强行 POST。**

可选路径（均需人工确认）：

1. **提高 leverage / margin / notional 上限**  
   例如 `leverage=30` 时 `required_margin ≈ 600/30 = 20`，但仍需确认 `required_notional` 不超过 `OKX_MAX_NOTIONAL_USDT`。

2. **改用最小名义价值更低的合约**  
   若 OKX 允许列表中有更小 `ctVal × minSz × price` 的 SWAP，调整 `OKX_ALLOWED_INST_IDS` 与 payload `symbol`（需同步 guard 配置）。

3. **先回到 OKX simulated trading**  
   设置 `OKX_SIMULATED_TRADING=true`，在模拟环境验证 open-close 流程与 reconcile，再考虑 live。

## 推荐 workflow

```bash
# 1. 只读预检（live mark price）
python scripts/check_okx_instrument_canary_feasibility.py

# 2. 仅当 SUITABLE 时生成 payload
python scripts/build_local_okx_live_canary_payload.py --close CURRENT_BTCUSDT_PRICE

# 3. POST webhook（source=local_canary），测试后删除 payload 文件
```

## 相关模块

## 边界

- 不修改 Binance 行为
- 不绕过 OKX Guard（仅前置拒绝不可行 payload）
- 不实现 TP/SL
- 不提交 `.env`、secret、payload、日志
