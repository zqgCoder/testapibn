# v6.5.2 OKX Live Rejection Guard

本版在 v6.5.1 OKX read-only adapter 基础上，增加 **OKX live rejection guard** 与 Dashboard preflight。

**不实现 OKX 下单、撤单、close、cleanup 写接口。**

## 目标

| # | 要求 |
|---|------|
| 1 | `EXCHANGE=okx` 时执行链路必须先经过 OKX guard |
| 2 | 拒绝时写入 journal：`status=okx_guard_rejected` + 明确 `skip_reason` |
| 3 | 不调用 OKX 写接口 |
| 4 | Dashboard 展示 OKX preflight / blocking_reasons |
| 5 | `EXCHANGE=binance` 行为不变 |

## OKX Guard 拒绝规则（按优先级）

| skip_reason | 条件 |
|-------------|------|
| `okx_readonly_mode` | `OKX_READONLY_MODE=true` |
| `okx_live_trading_disabled` | `OKX_LIVE_TRADING_ENABLED=false` |
| `okx_confirm_phrase_invalid` | 确认短语缺失或不匹配 |
| `okx_one_shot_required` | `OKX_REQUIRE_ONE_SHOT=true` 且无 active one-shot |
| `okx_symbol_not_allowed` | symbol/instId 不在 `OKX_ALLOWED_INST_IDS` |
| `okx_execution_not_implemented` | 以上均通过时仍拒绝（v6.5.2 无下单实现） |

## 执行顺序（EXCHANGE=okx）

```
POST /tradingview
  → runtime lock 检查
  → validate_okx_guard_before_plan()
  → journal: okx_guard_rejected
  → 无 OKX/Binance 下单
```

## Dashboard

`GET /dashboard/api/okx-canary/preflight?token=...`

返回字段：

- `exchange=okx`
- `readonly_mode`
- `okx_live_trading_enabled`
- `okx_confirm_phrase_configured` / `okx_confirm_phrase_valid`
- `allowed_inst_ids`
- `canary_ready`（v6.5.2 恒为 `false`）
- `blocking_reasons`

Runtime Control 页面在 `EXCHANGE=okx` 时显示 **OKX Live Rejection Preflight** 面板。

## 本地受控 rejection 测试

```bash
python scripts/build_local_okx_reject_payload.py --close CURRENT_BTCUSDT_PRICE
```

- `source=local_canary`
- `symbol=BTCUSDT` → 内部映射 `BTC-USDT-SWAP`
- 不打印 secret / payload 原文
- 输出 `_v652_okx_guard_reject_payload.json`（测试后删除）

POST（不 echo secret）：

```powershell
curl.exe -s -X POST "http://127.0.0.1:8000/tradingview" `
  -H "Content-Type: application/json" `
  --data-binary "@_v652_okx_guard_reject_payload.json"
```

预期 journal：

- `status=okx_guard_rejected`
- `skip_reason=okx_readonly_mode`（默认配置）

验收后确认：

- positions / open-orders / algo-orders 仍为空
- reconcile `error_count=0`
