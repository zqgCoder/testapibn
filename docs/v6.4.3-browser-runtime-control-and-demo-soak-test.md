# v6.4.3 Browser Runtime Control Dashboard

本版在 v6.4.2-stable 基线上新增浏览器 Runtime Control 控制面，用于 Binance demo 持续测试前的人工操作，不修改交易核心、Runtime Control 安全语义或 TradingView webhook 校验。

## 访问

1. 在 `.env` 中启用 Dashboard 并配置 Token：

```env
DASHBOARD_ENABLED=true
DASHBOARD_REQUIRE_TOKEN=true
DASHBOARD_TOKEN=your-dashboard-token

RUNTIME_CONTROL_ENABLED=true
RUNTIME_CONTROL_REQUIRE_TOKEN=true
RUNTIME_CONTROL_TOKEN=your-runtime-control-token
RUNTIME_STATUS_ALLOW_DASHBOARD_TOKEN=true
```

2. 浏览器打开（仅建议 localhost）：

```
http://127.0.0.1:8000/dashboard/runtime-control?token=YOUR_DASHBOARD_TOKEN
```

`RUNTIME_CONTROL_TOKEN` **不会**出现在 HTML、JS 或浏览器请求中；写操作由服务端 Dashboard API 代理，在进程内直接调用 `runtime_control` / `trader`。

## 页面功能

| 区域 | 说明 |
|------|------|
| 运行状态 | 显示 enabled / locked / effective_locked / one_shot 等字段，每 5 秒自动刷新 |
| Lock | 二次确认，默认 reason=`manual browser lock from dashboard` |
| One-Shot Unlock | TTL 30–3600（默认 300），二次确认；**无普通长期 Unlock 按钮** |
| BTCUSDT 状态 | 持仓 / 条件单 / 普通委托（每 10 秒刷新，可手动刷新） |
| Close / Cleanup | 二次确认 + 危险样式；仅 demo 维护接口 |
| Reconcile | `POST /dashboard/api/runtime-control/reconcile/run`，展示 summary |
| Journal | 最近 5 条 executions |

## 新增路由

### 页面

- `GET /dashboard/runtime-control`

### Dashboard API（需 `X-Dashboard-Token` 或 `?token=`）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/dashboard/api/runtime-control/status` | 已有，只读 |
| GET | `/dashboard/api/runtime-control/events` | 已有，只读 |
| POST | `/dashboard/api/runtime-control/lock` | 代理锁定 |
| POST | `/dashboard/api/runtime-control/unlock-once` | 代理 one-shot |
| GET | `/dashboard/api/runtime-control/symbol/{symbol}/positions` | 单币对持仓 |
| GET | `/dashboard/api/runtime-control/symbol/{symbol}/algo-orders` | 条件单 |
| GET | `/dashboard/api/runtime-control/symbol/{symbol}/open-orders` | 普通委托 |
| POST | `/dashboard/api/runtime-control/symbol/{symbol}/close` | 平仓维护 |
| POST | `/dashboard/api/runtime-control/symbol/{symbol}/cleanup` | 清理挂单 |
| POST | `/dashboard/api/runtime-control/reconcile/run` | 手动审计 |

顶层 `/runtime/unlock` 仍存在，但**未在浏览器页面暴露**。

## Demo soak test 建议流程

1. 打开 `/dashboard/runtime-control?token=...`，确认 `effective_locked=true`
2. One-Shot Unlock（TTL 300），确认 `one_shot.remaining=1`
3. 从 TradingView 或 curl 发送一条合法 TV 信号
4. 确认 journal 为 `protected` 或明确 Binance 错误（非 Python 异常）
5. 确认 runtime 自动回锁
6. 页面执行 Close → Cleanup → Reconcile，确认持仓/挂单为空、summary 干净
