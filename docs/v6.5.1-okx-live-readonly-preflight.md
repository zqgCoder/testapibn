# v6.5.1 OKX Live Read-Only Preflight

本版在 v6.5.0 `ExchangeClient` 抽象层上增加 **OKX live 只读预检**，**不实现 OKX 下单**。

## 目标

- `EXCHANGE=okx` 时通过 OKX REST API 只读查询持仓、普通委托、条件单
- Runtime Control Dashboard 复用现有查询接口（输入 `BTCUSDT` 内部映射为 `BTC-USDT-SWAP`）
- `Trader.execute()` 在 OKX 模式下拒绝执行，返回 `skip_reason=okx_execution_not_implemented`
- `EXCHANGE=binance` 行为与 v6.5.0 完全一致

## 硬性边界（本版未实现）

- OKX `place_order` / 实盘平仓 / 实盘撤单
- `cancel_open_orders` / `cancel_algo_orders` → 抛出 `OkxReadOnlyUnsupportedError`
- `close_position` → 返回 `status=okx_readonly_not_supported`（不下单）
- 不修改 Binance demo/live 交易路径（仍走 `self.client`）
- 不修改 Runtime one-shot / Live Guard / TradingView payload 核心逻辑

## 配置

| 变量 | 默认 | 说明 |
|------|------|------|
| `EXCHANGE` | `binance` | `binance` 或 `okx` |
| `OKX_API_KEY` | 空 | OKX API Key |
| `OKX_API_SECRET` | 空 | OKX API Secret |
| `OKX_API_PASSPHRASE` | 空 | OKX Passphrase |
| `OKX_BASE_URL` | `https://www.okx.com` | REST 根地址 |
| `OKX_SIMULATED_TRADING` | `false` | `true` 时加 `x-simulated-trading: 1` |
| `OKX_INST_TYPE` | `SWAP` | 永续合约类型 |
| `OKX_ALLOWED_INST_IDS` | BTC/ETH/SOL SWAP | reconcile 检查列表 |
| `OKX_READONLY_MODE` | `true` | v6.5.1 必须为 `true` |
| `OKX_LIVE_TRADING_ENABLED` | `false` | v6.5.1 必须为 `false` |

`EXCHANGE=okx` 时 `validate_runtime()` 要求配置 OKX 三项凭证。

## OKX 签名

Header:

- `OK-ACCESS-KEY`
- `OK-ACCESS-SIGN`
- `OK-ACCESS-TIMESTAMP`
- `OK-ACCESS-PASSPHRASE`

```
sign = Base64(HMAC_SHA256(secret, timestamp + METHOD + requestPath + body))
```

- `METHOD` 大写
- GET 无 body 时 `body=""` 
- query string 计入 `requestPath`（如 `/api/v5/account/positions?instId=BTC-USDT-SWAP`）
- timestamp：UTC ISO8601 毫秒，如 `2020-12-08T09:08:57.715Z`
- 仅 `OKX_SIMULATED_TRADING=true` 时设置 `x-simulated-trading: 1`

实现：`app/okx_auth.py`、`app/exchanges/okx.py`

## 只读 API

| ExchangeClient 方法 | OKX REST |
|---------------------|----------|
| `get_positions` | `GET /api/v5/account/positions?instId=...` |
| `get_open_orders` | `GET /api/v5/trade/orders-pending?instType=SWAP&instId=...` |
| `get_algo_orders` | `GET /api/v5/trade/orders-algo-pending?ordType=conditional&instType=SWAP&instId=...` |
| `reconcile` | 对 `OKX_ALLOWED_INST_IDS` 只读汇总 |

`reconcile` summary 字段：

- `open_position_count`
- `residual_order_symbol_count`
- `error_count`
- `warn_count`

## Dashboard

`/dashboard/runtime-control` 查询接口改走 `trader.exchange_client`：

- `GET .../symbol/BTCUSDT/positions`
- `GET .../symbol/BTCUSDT/open-orders`
- `GET .../symbol/BTCUSDT/algo-orders`
- `POST .../reconcile/run` → `exchange_client.reconcile()`

**无新增下单按钮。** Close/Cleanup 在 OKX 模式下不会提交实盘订单。

## Trader 拒绝执行

```python
# EXCHANGE=okx
skip_reason = "okx_execution_not_implemented"
```

在 Runtime lock / Live Guard 之前返回，不调用 Binance `self.client` 下单。
