## Validation Rules

TradingView payload validation now checks:

- position_strategy is required
- position_strategy must be replace / reject / add
- tps must be a non-empty array
- each TP price must be greater than 0
- each TP qty_pct must be in (0, 1]
- TP qty_pct sum must be within the configured range
- TP count must not exceed the configured max
- sent_at / timestamp / time can be parsed when present
- expired signals can be rejected when configured
- BUY signals require SL below reference price and TP above reference price
- SELL signals require SL above reference price and TP below reference price
- SL / TP minimum distance is enforced
- duplicate signal_id is recognized and skipped
- Runtime locked behavior remains safe

## Safety Behavior

Dirty TradingView payloads must be rejected as business-level failures, not server errors.

Expected rejection response:

- success false
- skipped true
- skip_reason tv_payload_invalid
- clear message
- signal_id included
- error_fields included when available

No rejected signal is allowed to:

- open a position
- close a position
- cancel orders
- unlock Runtime
- modify Runtime state

## Fixed Blocking Issue

Initial v6.4.0 testing found that invalid position_strategy caused:

- HTTP 500 Internal Server Error

Root cause:

- main.py called apply_position_strategy before validation
- app/tv_production.py raised an uncaught ValueError for invalid strategy

Fix:

- validate before mapping
- apply_position_strategy no longer leaks uncaught exceptions
- main.py catches strategy mapping errors and converts them into tv_payload_invalid business rejection
- journal records tv_sandbox_rejected with skip_reason tv_payload_invalid
