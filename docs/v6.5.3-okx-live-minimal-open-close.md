# v6.5.3 OKX Live Minimal Open-Close Canary

本版在 v6.5.2 OKX live rejection guard 基础上，实现 **极小金额 market 开多 → 立即 market 平仓**。

**不实现 TP/SL protected，不修改 Binance 行为。**

## 硬性边界

| # | 要求 |
|---|------|
| 1 | 仅 `source=local_canary` |
| 2 | `OKX_READONLY_MODE=false` |
| 3 | `OKX_LIVE_TRADING_ENABLED=true` |
| 4 | `OKX_CONFIRM_PHRASE` 匹配期望值 |
| 5 | Runtime one-shot active |
| 6 | `instId` ∈ `OKX_ALLOWED_INST_IDS`（BTCUSDT → BTC-USDT-SWAP） |
| 7 | 极小 `risk/margin/notional` 限额 |
| 8 | 仅 `side=buy` 开多，立即 close |
| 9 | 无 TP/SL 下单 |
| 10 | close 后 reconcile `error_count=0` |

## 配置

| 变量 | 默认 | 说明 |
|------|------|------|
| `OKX_TD_MODE` | `isolated` | `isolated` 或 `cross` |
| `OKX_MAX_RISK_USDT` | `1` | canary risk 上限 |
| `OKX_MAX_MARGIN_USDT` | `20` | canary margin 上限 |
| `OKX_MAX_NOTIONAL_USDT` | `100` | canary 名义价值上限 |

## 执行流程

```
POST /tradingview (source=local_canary, side=buy)
  → runtime lock 检查
  → OKX guard（配置 + source + side + 限额）
  → 读取 instrument metadata（lotSz/minSz/ctVal）
  → 计算最小合约张数 sz
  → place_market_order_minimal (buy)
  → close_position_market (close-position API)
  → reconcile
  → journal: okx_canary_completed
```

## OKX 合约张数

OKX `sz` 为**合约张数**，不得复用 Binance BTC 数量。

```
notional_usdt ≈ sz × ctVal × mark_price
```

使用 `GET /api/v5/public/instruments` 读取 `lotSz` / `minSz` / `ctVal`，取满足 `minSz` 的最小张数。

## 写接口（OkxExchange）

| 方法 | OKX API |
|------|---------|
| `place_market_order_minimal()` | `POST /api/v5/trade/order` |
| `close_position_market()` | `POST /api/v5/trade/close-position` |

## 本地 payload

```bash
python scripts/build_local_okx_live_canary_payload.py --close CURRENT_BTCUSDT_PRICE
```

前置：

- `EXCHANGE=okx`
- `OKX_READONLY_MODE=false`
- `OKX_LIVE_TRADING_ENABLED=true`
- 确认短语已配置且匹配

输出 `_v653_okx_live_canary_payload.json`（测试后删除，不提交 git）。

不打印 `WEBHOOK_SECRET` / payload 原文；仅打印 `signal_id`、`instId`、`side`、margin/leverage/sz 摘要。

Payload 使用 `risk_mode=manual`（无 SL/TP），`margin_usdt` + `leverage` 控制极小仓位。

## 手动步骤（概要）

1. Dashboard Runtime one-shot unlock
2. 生成 payload 并 `POST /tradingview`
3. 确认 journal `okx_canary_completed`
4. Dashboard 确认 positions/orders 为空
5. Reconcile `error_count=0`
6. Runtime lock
7. 删除临时 JSON

## Journal

| 场景 | status | skip_reason |
|------|--------|-------------|
| guard 拒绝 | `okx_guard_rejected` | `okx_*` |
| canary 成功 | `okx_canary_completed` | — |
