# v6.4.2 Backend TV Guard Price Normalization

## 问题原因

TradingView Pine 生成的 SL/TP 距离可能正好等于后端最小距离阈值（例如 0.25%）。由于浮点边界误差（例如 `distance=0.002499...` 对比 `min_stop_pct=0.0025`），后端会把合法方向的信号误判为 `tv_payload_invalid` 并拒绝。

Pine 策略会持续迭代，不应要求 Pine 精确适配后端阈值。后端作为统一入口，应在方向正确但距离不足时自动规范化 SL/TP。

## 自适应规则

以 payload 中的 `close` 作为 `ref_price`（不改用 ticker 或其他字段做规范化）。

当 `TV_SIGNAL_AUTO_NORMALIZE_GUARD_PRICES=true` 且 SL/TP 方向正确但距离 `<= MIN_PCT + EPSILON` 时：

| side | 字段 | 目标距离 |
|------|------|----------|
| BUY | SL | `ref * (1 - (MIN_STOP + BUFFER)/100)` |
| BUY | TP | `ref * (1 + (MIN_TP + BUFFER)/100)` |
| SELL | SL | `ref * (1 + (MIN_STOP + BUFFER)/100)` |
| SELL | TP | `ref * (1 - (MIN_TP + BUFFER)/100)` |

默认：`MIN=0.25%`，`BUFFER=0.05%`，规范化后目标距离至少 `0.30%`。

方向错误（例如 SELL 但 SL < close）**不会**被调整，仍由 `validate_sl_tp_levels` 拒绝。

## 配置项

| 变量 | 默认 | 说明 |
|------|------|------|
| `TV_SIGNAL_AUTO_NORMALIZE_GUARD_PRICES` | `true` | 启用自动规范化 |
| `TV_SIGNAL_GUARD_DISTANCE_BUFFER_PCT` | `0.05` | 在最小距离上额外增加的百分点 |
| `TV_SIGNAL_DISTANCE_EPSILON` | `0.000001` | 距离边界浮点容差 |

## 处理流程

1. `/tradingview` 基础字段校验（position_strategy、timestamp、tps qty 等）
2. `normalize_tv_guard_prices()` 调整 `sl` / `tps[].price`
3. `apply_position_strategy()` 映射策略
4. Pydantic 校验 + `validate_tv_policy()` 距离/方向复核
5. Runtime lock / one-shot / trader 执行（逻辑未改）

## Journal 追踪

执行结果 `result_json` 中包含：

```json
"tv_price_normalization": {
  "enabled": true,
  "adjusted": true,
  "reason": "guard_price_distance_too_close",
  "ref_price": 58312.73,
  "min_stop_pct": 0.25,
  "min_tp_pct": 0.25,
  "buffer_pct": 0.05,
  "adjustments": [...]
}
```

未调整时：`{"enabled": true, "adjusted": false}`

## 安全边界（未修改）

- Runtime Control / one-shot 语义不变
- Runtime locked 仍 `blocked_by_runtime_lock`
- `WEBHOOK_SECRET` 校验不变
- demo/testnet live endpoint 拒绝不变
- `position_strategy` 必填不变
- `tps.qty_pct` 合计校验不变

## 本地测试

```bash
python -m compileall main.py app
python -c "from app.trader import Trader; print('import ok')"
```

Python 单测示例（SELL + 0.25% 边界）：

```python
from app.config import Settings
from app.tv_production import normalize_tv_guard_prices, validate_sl_tp_levels
from app.schemas import TradingViewSignal

settings = Settings(_env_file=None, WEBHOOK_SECRET="x"*30)
payload = {
    "source": "tradingview",
    "signal_id": "TV-test-norm",
    "symbol": "BTCUSDT",
    "side": "sell",
    "close": 58312.73,
    "sl": 58458.51,
    "tps": [{"price": 58166.95, "qty_pct": 1.0}],
    "position_strategy": "replace",
}
normalized, meta = normalize_tv_guard_prices(payload, settings)
assert meta["adjusted"] is True
signal = TradingViewSignal.model_validate({**normalized, "secret": "x"*30, "entry_type": "market", "risk_mode": "fixed_usdt", "risk_usdt": 3, "margin_usdt": 80})
assert validate_sl_tp_levels(normalized, signal, settings, None) is None
```

方向错误 SL 仍应被拒绝；Runtime locked 下规范化后的合法信号 journal 状态为 `blocked_by_runtime_lock`。
